简单工厂模式(又叫静态工厂模式) :
    no_factory : 没有使用简单工厂,看起来似乎也不错,但是却有个很大的问题 -- 可扩展性不好
                当前只有两种pizza,当我们需要在添加一种pizza的时候,会发现除了需要新建一个Pizza的子类外,
                还需要修改商店类(OrderPizza)类种的逻辑,现实中会有多个OrderPizza类都在下单Pizza,那么就
                需要修改多处,映射在代码中,就是多个地方引用了OrderPizza类,对OrderPizza类的修改就可能导致问题,
                换句话说 :若有一个商店OrderPizza不再可以订购某种pizza时,就会导致其他可以的OrderPizza类异常

                ------->   引出简单工厂模式 - 将订购的方式抽到工厂中

问题 :
    1.当一个工厂不适合将所有的生产线都包含进去,那么就会出现多个工厂,那么每次商店就需要先判断去不同的工厂订购
        如: 中国的工厂(ChinaPizza)中有chinaPizza,外国的工厂有其他pizza,这种情况商店需要去不同的工厂,
            就要创建不同的工厂对象
